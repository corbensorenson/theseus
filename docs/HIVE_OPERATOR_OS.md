# Hive Operator OS

The Hive Operator OS is the usability/control layer that ports the useful
Hermes, OpenClaw, MoECOT, CCA, and BugBrain patterns into Project Theseus Hive.
It is generated by:

```powershell
python scripts\hive_operator_os.py --out reports\hive_operator_os.json --markdown-out reports\hive_operator_os.md
```

The source contract is `configs/hive_operator_os.json`.

## What It Adds

- one shared command vocabulary for dashboard, mobile, CLI, tray/menu bar,
  relay, and future chat channels;
- a durable SQLite Hive work board at `reports/hive_work_board.sqlite`;
- background-task, watch, wake, and phone-delivery contracts;
- persistent goal visibility with budgets, judge, current step, and stop
  conditions;
- first-party Hive skill packs with dynamic loading and hygiene metrics;
- before/after tool-hook contracts plus a live hook ledger for the VIEA action
  executor and Hive work-board executor;
- a live command-channel proof path for `/background run broad transfer status`;
- a high-transfer curriculum lane that schedules source-agnostic concepts
  instead of benchmark-name overfitting;
- rollback, checkpoint, worktree, remote-control TTL, kill-switch, signed
  update, and personality-charter safety contracts;
- feedback routing from corrections into memory, skills, residuals, and
  follow-up work.

## Command Vocabulary

The first shared commands are:

`/status`, `/goal`, `/background`, `/watch`, `/wake`, `/board`, `/skill`,
`/rollback`, `/takeover`, `/bench`, `/train`, `/schedule`, `/teacher`,
`/pause`, `/resume`, `/stop`, and `/feedback`.

Every channel adapter should translate its local UI into this vocabulary rather
than inventing a parallel control plane.

## Hive Skills

Hive Skills are user-facing apps over lower-level tools. The initial registry
includes Mail, Calendar, Files, Browser, GitHub, Linear, Notes/Obsidian, Daily
Brief, Repo Repair, Remote Desktop, Training Status, Benchmark Watch, Cloud
Compute, Hive Updates, and Personality Memory.

Skills expose permissions, platform/status, load terms, usage count, inferred
success rate, and hygiene state. New skills should update this registry before
becoming a new independent tool.

## Work Board

The work board currently ingests:

- VIEA feedback actions;
- Hive task queue rows;
- Hive scheduler placements;
- utilization manager planned/executed actions;
- training orchestrator jobs;
- the latest autonomous goal;
- Vacation Mode repair actions;
- broad-transfer gap work;
- high-transfer curriculum tasks.

Reports remain views. The SQLite board is the durable local substrate for task
status, evidence, comments, dependencies, and events.

The executor path is:

```powershell
python scripts\hive_work_board_executor.py --execute --resume --max-tasks 1 --max-steps 1
```

The first live command channel is:

```powershell
python scripts\hive_work_board_executor.py --command-text "/background run broad transfer status" --execute --max-tasks 1
```

This creates a board task, assigns it to the current best node, emits hook
ledger entries, writes run evidence, and marks the task `done`, `failed`, or
`blocked`.

High-transfer board tasks are now executable directly. The board can run
`type_and_return_shape`, `edge_conditions`, `multi_turn_conversation`,
`repo_repair`, and `long_horizon_tool_use` under the same hooks, node-placement
rules, and improvement contract. Completed generated tasks are preserved across
Operator OS refreshes by task history, so scheduler refreshes do not erase
finished work.

The dashboard's Operator OS panel also reads the morning report so a phone,
browser, tray, or future chat channel can answer the overnight question:

```text
What improved, what failed, what Theseus learned, what needs approval, and what runs next?
```

The panel also shows the node registry and overnight proof. The important
operator split is:

- `training_eligible_node_count`: safe heavy-training nodes.
- `light_eligible_node_count`: trusted nodes that can still run chat,
  status, indexing, and light inference.
- `best_inference_node`: where weak clients or phone chat should route.
- `best_training_node`: where checkpoint-producing learning should run.

This lets a Mac with low free disk remain useful without accidentally treating
it as a safe training host.

## Safety

The operator layer does not weaken autonomy gates. It makes them visible:

- public benchmark data remains calibration-only;
- teacher use remains architecture diagnosis only;
- remote control requires TTL, ledger, provider readiness, and a kill switch;
- mutating code/self-update/training-config actions require checkpoint or
  worktree isolation;
- node updates converge through signed/versioned update reports;
- the personality core is treated as a durable charter plus audited adaptive
  runtime context.

## Dashboard

The dashboard has a `Hive Operator OS` panel with channel parity, board state,
node registry, overnight proof, background tasks, persistent goals, skill
hygiene, hook coverage, and execution safety. Press `Refresh` to regenerate the
reports, `Run Board Step` to execute one bounded task, `High-Transfer` to
refresh concept-transfer tasks, or submit the `/background run broad transfer
status` command from the command box.
